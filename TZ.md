# Техническое задание на проект: Система отслеживания публичных утечек данных для IT-администраторов с целью выявления компрометации корпоративных доменов

## 1. Цель проекта
Создание веб-сервиса для оперативного выявления фактов компрометации корпоративных доменов через проверку публичных баз утечек данных. Система помогает IT-администраторам централизованно отслеживать, не попали ли учётные записи сотрудников в известные инциденты, и формировать отчёты для усиления защиты.

## 2. Роли пользователей
- **Гость**: просмотр общей статистики (количество проверенных доменов, распределение по статусам риска).
- **IT-администратор**: добавление доменов в реестр, запуск проверки через HaveIBeenPwned API, просмотр детальных отчётов, экспорт результатов.
- **Суперпользователь (Django admin)**: управление пользователями, очистка устаревших записей, настройка политик хранения данных.

## 3. Модели данных (сущности)
### CorporateDomain
Реестр корпоративных доменов компании.
Поля: `name` (unique), `owner`, `criticality` (Low/Medium/High), `is_active`.

### BreachRecord
Факты обнаруженных утечек по домену.
Поля: `domain` (FK → CorporateDomain), `breach_name`, `breach_date`, `data_classes`, `accounts_count`.

### VerificationLog
Журнал выполненных проверок для аналитики.
Поля: `domain` (FK → CorporateDomain), `checked_at`, `status` (Clean/Warning/Critical), `breaches_found`, `risk_comment`.

*Все модели связаны через ForeignKey. BreachRecord и VerificationLog не могут существовать без CorporateDomain.*

## 4. Ключевой функционал (User Stories)
1. Администратор добавляет домен через форму → система валидирует формат → сохраняет в БД.
2. Администратор запускает проверку → система запрашивает HaveIBeenPwned API → парсит ответ → создаёт записи в BreachRecord → обновляет статус в VerificationLog.
3. На дашборде отображается таблица доменов с цветовой индикацией риска и график Plotly: динамика количества утечек по месяцам.
4. Администратор может выгрузить результаты проверки в CSV для служебной записки.

## 5. Внешние интеграции / Аналитика
- **API**: HaveIBeenPwned (k-anonymity mode) для проверки email-адресов домена.
- **Аналитика**: `pandas` для агрегации метрик, `plotly.express` для отрисовки интерактивных диаграмм.
