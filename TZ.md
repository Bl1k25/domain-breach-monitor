Техническое задание: Система Threat Intelligence для мониторинга корпоративных доменов
1. Цель проекта
Создание веб-сервиса для мониторинга угроз, связанных с корпоративными доменами, с использованием платформы Intelligence X. Система позволяет IT-администраторам оперативно выявлять утечки конфиденциальных данных (файлы cookies, логи вредоносного ПО, пасты), связанные с инфраструктурой компании, анализировать риск компрометации и принимать меры по защите.
2. Роли пользователей

    Гость: просмотр аналитики (общая статистика рисков, графики распределения угроз по типам).
    IT-администратор: управление реестром доменов (добавление/удаление), запуск сканирования через Intel X API, просмотр детальных отчётов об угрозах (включая просмотр содержимого файлов).
    Суперпользователь (Django Admin): управление пользователями, полный контроль над моделями данных.

3. Модели данных (сущности)
CorporateDomain
Реестр контролируемых доменов.

    Поля: name (доменное имя, уникальное), owner (ответственный), criticality (уровень важности), is_active.

VerificationLog
Журнал всех проведенных проверок.

    Поля: domain (FK), checked_at (дата проверки), status (Clean/Warning/Critical на основе X-Score), risk_comment.

ThreatDetail
Детализация конкретных найденных угроз (записей из Intel X).

    Поля: verification (FK), system_id (ID записи в Intel X), storage_id, xscore (релевантность 0-100), bucket (источник, например leaks.logs), media_type (тип файла), size, date_found.
    Связь: Принадлежит к VerificationLog.

BreachRecord
Агрегированная запись об инциденте (для общей статистики).

    Поля: domain (FK), breach_name, source (Intel X), threat_score.

4. Ключевой функционал (User Stories)

    Управление реестром: Администратор добавляет домен через форму с валидацией формата. Администратор может удалить домен (реализовано через каскадное удаление связанных логов и угроз).
    Интеграция с Intel X: Запуск проверки инициирует двухэтапный запрос к API:
        Отправка селектора (домена) на поиск (/intelligent/search).
        Получение ID поиска и извлечение результатов (/intelligent/search/result).
    Анализ содержимого: При просмотре детали угрозы система определяет тип файла. Для текстовых данных отображается содержимое, для бинарных файлов (например, SQLite базы браузеров) — отображается HEX-дамп или sanitized-текст.
    Визуализация: На дашборде доступны графики распределения найденных угроз по источникам (buckets) и уровням риска.

5. Внешние интеграции и Стек

    API: Intelligence X API v5.
        Методы: /intelligent/search, /file/preview (для просмотра содержимого утечек), /file/view.
        Особенности: Обработка ошибок 401/402/404, корректная работа с кодировкой и бинарными данными.
    Backend: Python, Django.
    Аналитика: pandas для обработки метрик, plotly для интерактивных графиков.
    Frontend: Bootstrap 5, AJAX для динамической подгрузки содержимого файлов (модальные окна).
